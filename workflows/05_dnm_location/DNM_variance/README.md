This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

This project tests whether inter-family and inter-individual variance in autosomal de novo mutation (DNM) rates exceeds what is expected under a Poisson null model, across 7 spider species (AFR, BIC, DUM, LIN, MIM, SAR, TEN). The statistical approach is Monte Carlo simulation: observed variance is compared to a null distribution generated by drawing per-individual DNM counts from Poisson(callable_sites × species_rate).

## Running the Workflow

All GWF commands must be run from `dnm_variance/` (the directory containing `workflow.py`):

```bash
conda activate gwf_new
cd dnm_variance/

gwf status                        # check all 14 targets (7 species × 2 analyses)
gwf run                           # submit all pending targets
gwf logs --stderr {target_name}   # inspect SLURM stderr for a failed target
```

Target naming convention: `{SP}_dnm_variance_simulation` and `{SP}_dnm_individual_variance_simulation` where `{SP}` is uppercase (e.g. `AFR_dnm_variance_simulation`).

## Architecture

### Workflow Pattern
The codebase mirrors the sibling `DNM_location` workflow. The three-layer GWF pattern is:
- `workflow.py` — globs configs, calls `dnm_variance_workflow()` for each species
- `workflow_sources.py` — parses YAML, wires up two targets per species (family + individual)
- `workflow_templates.py` — builds the SLURM shell spec; returns `(inputs, outputs, options, spec)`

Note: templates return a plain tuple `(inputs, outputs, options, spec)`, not `AnonymousTarget`. The target is registered in `workflow_sources.py` via `gwf.target(...) << spec`.

### The Two Analyses
Both scripts share the same inputs and CLI signature. Key difference:

| | Family-level | Individual-level |
|---|---|---|
| Script | `dnm_variance_simulation.py` | `dnm_individual_variance_simulation.py` |
| Unit | Family (deduplicated across siblings) | Individual offspring |
| Dedup | `(chrom, pos, ref, alt)` per family group | `(chrom, pos, ref, alt)` per child |
| Authoritative list | `callable_per_ind` keys → family keys | `callable_per_ind` keys directly |

Both: include 0-DNM units (families/individuals absent from DNM file are counted as 0), apply population variance (`ddof=0`), one-tailed p-value (upper tail).

### Input Data
- `data/DNM/{SP}_minDP_26_automate_classified.tsv` — TSV with header; columns include `chrom, pos, ref, alt, child, father, mother, ...`
- `data/callable/{SP}_callable_minDP26.tsv` — TSV, no header; cols: `chrom`, `count`, `individual_chrom` where `individual_id = col3[:-(len(chrom)+1)]`
- Both are symlinks to upstream pipeline outputs at `steps/germline/minDP/`

### Individual ID Parsing
`child` values look like `AFR_family1_S3_offspring`. Family key is derived via `child.rsplit('_S', 1)[0]` → `AFR_family1`.

### Conda Environment
- `python_phylo` — runs the analysis scripts (numpy, pandas)
- `gwf_new` — runs GWF workflow management

Shell specs must activate conda with:
```bash
CONDA_BASE=$(conda info --base)
source $CONDA_BASE/etc/profile.d/conda.sh
conda activate python_phylo
```
(`source activate` does not work on this system.)

## Configuration Files

Each species has `dnm_variance/configurations/{SP}.config.yaml`. Key parameters:

- `x_chromosomes` — list of sex chromosome names to exclude (LIN has 3: `lin_X1/X2/X3`; all others have 2)
- `excluded_trios` — list of offspring IDs to exclude from both analyses; BIC currently excludes all 12 offspring from family2 and family3
- `n_simulations: 1000`, `random_seed: 42`

SAR note: `SAR_family3` has 0 DNMs in the DNM file but 6 callable offspring — it is treated as a valid 0-DNM family by default. Add its offspring to `excluded_trios` in `SAR.config.yaml` to exclude it.

## Output Structure

```
steps/dnm_variance/{SP}/
  {sp}_variance_simulations.tsv           # family-level: simulation_id, simulated_variance
  {sp}_variance_summary.txt               # family-level: per-family breakdown + p-value
  {sp}_individual_variance_simulations.tsv
  {sp}_individual_variance_summary.txt
logs/
  {sp}_dnm_variance_simulation.DONE
  {sp}_dnm_individual_variance_simulation.DONE
```
